# Fatbeats-content learn — never solve the same thing twice

1. Write the decision to `knowledge/decisions/YYYY-MM-DD-{slug}.md`:
   **Decision** (what was chosen/rejected), **Why** (her actual words if
   available), **Rule?** (does this generalize beyond this release?).
2. If it generalizes, fold it in the same turn:
   - editing philosophy → `knowledge/editing-rules.md` (hard rule vs
     preference — be honest about which);
   - format behavior → `knowledge/release-types.md`;
   - caption voice → note under a "Voice" heading in the decision file,
     and reference it from `fatbeats-content-caption` workflows.
3. If the same fix has now happened twice in library code or a skill,
   flag it for `fatbeats-content-reflect` (system change), not just knowledge.
4. Confirm the release record reflects the decision (e.g. winning variant
   noted in the release README, status advanced, type confirmed flag).

A lesson that isn't written down is a lesson we pay for twice.
